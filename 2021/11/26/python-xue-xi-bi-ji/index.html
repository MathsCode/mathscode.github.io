1.2.4 列表元素的排序

'''
    sort()函数
    1.在原列表的基础上进行的，不会产生的新的列表
    2.默认排序规则是从小到大

new_list = sorted(lst)

new_list = sorted(lst,reverse=True)

1.3 字典

  • 以==键值对==的方式存储数据，字典是一个无序序列
  • 存储时要先经历==哈希函数==的计算后，再存储入字典当中
  • ==学习下来感觉倒是跟c语言的数组的使用方式蛮像的，更像c++中的map容器==

    获取字典视图的三种方法

1.3.3 字典生成式

  • 已知两个列表，第一个列表为字典中的键，第二个列表作为字典中的值，生成这样的一个列表
  • 使用内置函数zip()列表中的元素打包成元组
  • 语法格式（类似于列表生成式）：
  • {字典key表达式 : 字典value表达式 for key变量, value变量 in zip(可迭代对象1，可迭代对象2)}
items = ['fruits','books','others']
prices = [96,97,98]
lst = zip(items,prices)
print(list(lst))            # [('fruits', 96), ('books', 97), ('others', 98)]

d = {i:j for i,j in zip(items,prices)}
print(d)                    # {'fruits': 96, 'books': 97, 'others': 98}
print(d.items())            # dict_items([('fruits', 96), ('books', 97), ('others', 98)])
# 字母变大写
d = {i.upper():j for i,j in zip(items,prices)}
print(d)                    # {'FRUITS': 96, 'BOOKS': 97, 'OTHERS': 98}

1.4.2 将元组设置为不可变序列的原因

  • 在多任务的环境下，在对某个对象有多个同时操作时不需要加锁
  • 元组中存储的是对对象的引用
  • 如果元组中的对象本身就是不可变对象，则不能再修改其引用对象
  • 如果元组中的对象本身是可变对象，则可以对其数据进行更改

1.5 集合

  • 与列表、字典一样都是可变类型的序列
  • 集合中的元素不允许重复，自动去除重复数据
  • ==集合中的元素是无序的，（列表是有序的），这里的有序是指操作过程中不改变初始的顺序，无序是指操作过程可能会改变初始的顺序==
  • 集合中只有key 没有 value
  • 底层数据结构是哈希表

    1.5.1 集合的创建方式

'''集合的创建'''
s = {1,2,3,3,4,4,5,5,6,'python','xjm','c++','python'}

s = set(range(6))
print(s)                            # {0, 1, 2, 3, 4, 5}
# 利用set将列表/元组/转变成集合，去除列表中的重复数据
lst = ['python','c++','java','js','c++','python']
print(set(lst))                     # {'c++', 'js', 'python', 'java'}改变了顺序
print(set(('python','c++','ttt')))  # {'ttt', 'c++', 'python'},改变了顺序

1.5.4 集合的数据操作

操作 语法
交集 s1.intersection(s2) 或 s1 & s2
并集 s1.union(s2) 或 s1|s2
差集 s1.difference(s2) 或 s1-s2
对称差集 s1.symmetric_difference(s2) 或 s1^s2 
'''求交集的两种方式，不改变初始集合'''
s1 = {10,20,30,40}
s2 = {10,30,50,70}
print(s1.intersection(s2))      # {10, 30}
print(s1 & s2)                  # {10, 30}

'''求并集的两种方式，不改变初始集合'''
print(s1.union(s2))             # {70, 40, 10, 50, 20, 30}
print(s1|s2)                    # {70, 40, 10, 50, 20, 30}

'''求差集的两种方式，不改变初始集合'''
print(s1.difference(s2))        # {40, 20}
print(s1-s2)                    # {40, 20}

print(s2.difference(s1))        # {50, 70}
print(s2-s1)                    # {50, 70}

'''求对称差集（两集合的并减交）的两种方式，不改变初始集合'''
print(s1.symmetric_difference(s2))  # {70, 40, 50, 20}
print(s1^s2)                    # {70, 40, 50, 20}

1.6 列表、字典、元组、集合总结

数据结构 是否可变 是否重复 是否有序 定义符号
列表（list） 可变（有增删改操作） 元素可重复 存储顺序在操作过程中不发生变化，即有序 [ ]
元组（tuple） 不可变（无增删改操作） 元素可重复 存储顺序在操作过程中不发生变化，即有序 ( )
字典（dict） 可变（有增删改操作） key不可重复，value可重复 存储顺序在操作过程中会发生变化，即无序 {key:value}
集合（set） 可变（有增删改操作） 元素不可重复 存储顺序在操作过程中会发生变化，即无序 { }

  • 可由’python’ “python” ‘’’python’’’定义，这三者定义的是同一个对象（id相同）
  • 对于相同的字符串只保留一份拷贝，后续创建相同字符串时，不会开辟新的空间，而是只是把该字符串的地址赋给新创建的变量，这叫做Python的字符串驻留机制

1.8.2 函数的参数调用

  • 与c语言类似，python函数中的参数也有形参和实参，定义一样。
  • 实参有：位置实参（根据形参对应的位置进行实参传递），关键字实参（根据形参名进行实参传递）
def fun(a,b):
    print('a=',a)   # a= 10
    print('b=',b)   # b= [10, 20, 30, 40]
    a = 100         
    b.append(10)    
    print('a=',a)   # a= 100
    print('b=',b)   # b= [10, 20, 30, 40, 10]

t1 = 10
t2 = [10,20,30,40]
print(t1)           # 10
print(t2)           # [10,20,30,40]
fun(t1,t2)
print(t1)           # 10
print(t2)           # [10,20,30,40,10]

==这个程序中，函数fun中对形参a,b的值都采取了改变操作，为什么对应的实参t1,t2只有t2发生了改变，而t1却没有发生改变呢？==

因为python中万物皆对象，当程序中对t1,t2 “ 赋了相应的值 ” 时，其实是开辟了值为10的一个对象和值为[10, 20, 30, 40]的一个对象，然后t1,t2分别指向了相应的对象。

而当t1,t2作为实参传入函数当中时，此时t1,t2仍然指着相应的对象，同时，a 也指向了 10 这个对象，b 也指向了[10, 20, 30, 40]这个对象，在函数体中的操作只是对a,b进行操作，对a重新赋值100时，是在内存中又开辟了100这样的一个对象，让a指向了它，此时说明，a与t1所指向的对象是不同的对象了，而对b指向的列表对象最后插入10元素，是对[10, 20, 30, 40]这个对象进行操作，使其变成了[10, 20, 30, 40,10]，但是t2仍然指的是这个对象，只不过对象的值发生了改变，所以就出现了t1未变化而t2发生了改变的情况

  • 如果是不可变对象，在函数体内部形参的修改不会影响到实参的值
  • 如果是可变对象，在函数体内对形参的修改会影响到实参的值

1.8.3 函数的返回值

  • 函数可以返回一个值（这里的值并非指一个数，可以是一个列表或一个集合等等）也可以返回多个值
  • 若返回多个值时，结果为==元组==
def fun(num):
    odd = []
    even = []
    for i in num:
        if(i%2):
            odd.append(i)
        else:
            even.append(i)
    return odd,even

lst = range(1,10,1)
print(fun(lst))         # ([1, 3, 5, 7, 9], [2, 4, 6, 8])
